# 高德地图 Key 配置说明

本文档用于说明本项目如何先接入高德真实地图，再继续做“常熟地域特色地图皮肤”的自定义调试。

## 1. 当前接入状态

项目已经预留并接入了高德地图 JS API 2.0：

- `src/map/amapLoader.ts`：读取环境变量并加载高德 JS API。
- `src/components/ChangshuMap.tsx`：判断是否配置了高德 Key。
- `src/components/AmapChangshuMap.tsx`：渲染真实高德地图、点位、卡片弹层和路线折线。
- `src/map/amapRouteService.ts`：将右侧行程卡片顺序转换成高德步行 / 骑行 / 驾车路线规划。
- `src/components/FallbackChangshuMap.tsx`：未配置高德 Key 或加载失败时使用本地轻量地图兜底。

也就是说：

- 配置 `VITE_AMAP_KEY` 后，页面会优先使用高德真实地图。
- 不配置 Key 时，页面仍然可以打开，使用本地 fallback 地图。
- 用户拖入右侧行程槽的卡片会同步成为路线点。
- 如果高德路线规划可用，会生成真实道路路径；否则保留预览路线。

## 2. 申请高德 Key

1. 打开高德开放平台并登录。
2. 进入「控制台」。
3. 创建应用。
4. 在应用下添加 Key。
5. Key 类型选择「Web 端（JS API）」或页面中等价的 Web JS API 类型。
6. 复制生成的 `Key`。
7. 复制该 Key 对应的安全密钥，也就是 `securityJsCode`。

官方文档说明：高德 JS API 2.0 正常调用前需要申请 Web 平台 JS API 的 Key 和安全密钥；2021-12-02 以后申请的 Key 需要配合安全密钥一起使用。

## 3. 本地开发配置

在项目根目录新建 `.env.local`，内容如下：

```env
VITE_AMAP_KEY=你的高德Web端JSAPIKey
VITE_AMAP_SECURITY_CODE=你的securityJsCode安全密钥
VITE_AMAP_STYLE=amap://styles/normal
```

注意：

- `.env.local` 已被 `.gitignore` 忽略，不要提交到仓库。
- Vite 只会在启动时读取环境变量，修改 `.env.local` 后需要重启开发服务器。
- 如果浏览器仍然显示本地 fallback 地图，优先检查 Key、安全密钥、域名白名单和浏览器控制台报错。

启动方式：

```bash
npm install
npm run dev
```

默认访问：

```text
http://localhost:5173/
```

## 4. 域名白名单建议

本地开发建议在高德控制台允许：

```text
localhost
127.0.0.1
```

如果后续部署到线上，还需要把正式域名加入允许范围，例如：

```text
changshu.example.com
www.changshu.example.com
```

具体字段名称以高德开放平台控制台当前页面为准。

## 5. 地图皮肤配置

当前项目用 `VITE_AMAP_STYLE` 控制高德底图样式。

使用官方样式：

```env
VITE_AMAP_STYLE=amap://styles/normal
VITE_AMAP_STYLE=amap://styles/whitesmoke
VITE_AMAP_STYLE=amap://styles/fresh
VITE_AMAP_STYLE=amap://styles/macaron
```

使用自定义地图样式：

1. 进入高德「自定义地图」平台。
2. 创建常熟文旅主题样式。
3. 调整水系、绿地、道路、建筑、POI 等元素视觉。
4. 发布样式。
5. 获取地图样式 ID。
6. 写入 `.env.local`：

```env
VITE_AMAP_STYLE=amap://styles/你的样式ID
```

重启开发服务器后，项目会在高德真实地图上应用新的视觉皮肤。

## 6. 后续自定义开发路径

建议按三层推进：

第一层：高德真实地图接入

- 完成 Key 配置。
- 确认常熟地图能快速加载。
- 确认四类点位显示正常。
- 确认右侧行程卡片能同步成路线点。
- 确认步行 / 骑行 / 驾车路线可生成真实道路路径。

第二层：高德样式皮肤

- 使用高德自定义地图平台调整底图颜色、道路层级、水系、绿地、建筑和 POI。
- 保留真实地理坐标、真实道路网络和真实缩放能力。
- 让底图呈现常熟水乡、虞山、尚湖、古城肌理等地域特征。

第三层：自定义覆盖物

- 保留高德底图和道路规划能力。
- 在上层叠加我们自己的四类点位 Marker。
- 在上层叠加卡片、行程槽、手绘层、路线状态。
- 后续图片素材交付后，替换卡片和点位素材，不影响底层地图能力。

## 7. 常见问题

页面还是轻量地图：

- 没有创建 `.env.local`。
- `.env.local` 变量名写错。
- 修改 env 后没有重启 `npm run dev`。
- 高德 Key 类型不是 Web JS API。
- 安全密钥未配置或不匹配。
- 当前访问域名没有加入高德控制台允许范围。

地图白屏或控制台报鉴权错误：

- 检查 `VITE_AMAP_KEY`。
- 检查 `VITE_AMAP_SECURITY_CODE`。
- 检查 Key 是否启用 JS API 2.0。
- 检查浏览器控制台里的高德错误码。

真实路线没有生成：

- 确认至少加入两个行程点。
- 确认点位数据中有 `lng` 和 `lat`。
- 确认已加载 `AMap.Walking`、`AMap.Riding`、`AMap.Driving` 插件。
- 确认高德服务没有因为 Key、配额或网络失败而返回错误。

## 8. 安全说明

当前 Demo 阶段使用前端明文配置 `securityJsCode`，便于快速开发和调试。

如果进入生产环境，建议改成官方推荐的代理服务方式：把安全密钥放在服务端，通过代理转发高德请求，避免把安全密钥长期暴露在浏览器端。
